# Contributing to RAG Runtime Kernel

Thank you for your interest in improving the RAG Runtime Kernel.

## How to Contribute

**This project accepts issue reports only.** We do not accept pull requests or code contributions at this time.

### Reporting Bugs

If you encounter unexpected behavior while using the system:

1. Check [existing issues](../../issues) to avoid duplicates
2. Open a new issue using the **Bug Report** template
3. Include your environment details (LLM platform, spec version, filesystem access)
4. Reference the relevant spec section (§) if you can identify it
5. Include RAG state information if available

### Requesting Features

If you have an idea for an enhancement:

1. Open a new issue using the **Feature Request** template
2. Describe the problem it solves
3. Note which platform(s) it applies to

### What Makes a Good Report

- **Specific:** Include exact error messages, state values, and steps to reproduce
- **Contextual:** Tell us which LLM platform and execution mode you're using
- **Referenced:** Point to the spec section (§0–§38) where the behavior diverges from expected

## What We Do With Reports

Every issue is reviewed and categorized. Validated bugs and approved enhancements feed directly into the next specification release. Contributors whose reports lead to spec changes are credited in the version history.
