## 1. Problem Statement

v3.1.x operates in two modes:

- **Claude Desktop (MCP-enabled):** Full filesystem access via Filesystem MCP. Atomic writes, hash verification, WAL — all enforced by the LLM self-executing the spec.
- **GPT Web (autonomous):** No filesystem access. All persistence requires manual user download/upload. WAL, atomic writes, hash verification — advisory only.

This gap means GPT Web users get a fundamentally weaker experience. The spec documents the constraints but cannot solve them from inside the LLM context window.

## 2. Solution: Local Runtime Bridge

A lightweight Python process running on the user's machine that:

1. Exposes a **localhost HTTP API** for filesystem operations
2. Implements the spec's persistence stack (atomic writes, WAL, hash verification) as **enforced code** rather than LLM self-discipline
3. Bridges to LLMs via:
   - **Claude Desktop:** MCP server protocol (stdio transport)
   - **GPT Web:** OpenAI Custom Actions (localhost HTTP via tunnel)
   - **Any LLM:** Direct HTTP calls from user-pasted code blocks

## 3. Design Principles

1. **Zero external dependencies.** Python 3.10+ standard library only. Single `pip install rag-runtime-kernel`.
2. **Filesystem remains source of truth.** The bridge is a transport layer, not a database.
3. **Spec-faithful.** Every operation maps 1:1 to a spec section.
4. **Fail-safe.** Bridge crash = LLM falls back to autonomous mode (v3.1.x). No data loss.
5. **Localhost-only by default.** No network exposure unless user configures a tunnel.

## 6. State Machine Engine

Transitions enforced as code:

```python
TRANSITIONS: dict[State, set[State]] = {
    State.BOOTING:        {State.READY, State.RECOVERY},
    State.READY:          {State.INGESTING, State.WORKING, State.CHECKPOINTING, State.CLOSING},
    State.INGESTING:      {State.READY, State.CHECKPOINTING, State.RECOVERY},
    State.WORKING:        {State.READY, State.CHECKPOINTING, State.RECOVERY},
    State.CHECKPOINTING:  {State.READY, State.CLOSING, State.RECOVERY},
    State.CLOSING:        set(),  # terminal
    State.RECOVERY:       {State.READY, State.BOOTING},
}

def transition(self, target: State) -> bool:
    if target not in TRANSITIONS[self.current]:
        self.wal.append(Event.INVALID_TRANSITION, self.current, target)
        return False
    self.wal.append(Event.TRANSITION, self.current, target)
    self.current = target
    return True
```

## 7. Persistence Engine

### 7.1 Atomic Writes (M-020)

```python
def atomic_write(path: Path, data: bytes) -> None:
    tmp = path.with_suffix('.tmp')
    bak = path.with_suffix('.bak')
    tmp.write_bytes(data)
    if hashlib.sha256(tmp.read_bytes()).hexdigest() != hashlib.sha256(data).hexdigest():
        tmp.unlink()
        raise WriteVerificationError(path)
    if path.exists():
        shutil.copy2(path, bak)
    tmp.replace(path)  # atomic rename
```

### 7.2 WAL with fsync (M-022)

```python
def append(self, event: Event, **kwargs) -> None:
    entry = {
        "seq": self.next_seq(),
        "ts": datetime.utcnow().isoformat() + "Z",
        "event": event.value,
        **kwargs
    }
    line = json.dumps(entry, separators=(',', ':')) + '\n'
    self.fd.write(line)
    self.fd.flush()
    os.fsync(self.fd.fileno())
```

### 7.3 Hash Verification at Boot (M-030)

```python
def verify_hashes(hot: dict) -> list[str]:
    errors = []
    for key in ('state_hash', 'inventory_hash'):
        stored = hot.get(key, '')
        if stored in ('', 'PENDING'):
            continue  # first boot — skip per GPT-W003 sentinel
        computed = compute_hash(hot, exclude_keys={key})
        if stored != computed:
            errors.append(f"{key}: mismatch")
    return errors
```

### 7.4 COLD Partition Manager (M-015, M-033)

```python
class ColdManager:
    def __init__(self, cold_dir: Path):
        self.cold_dir = cold_dir
        self.index: dict[str, Path] = self._build_index()
        self.loaded: dict[str, dict] = {}

    def get(self, partition: str) -> dict:
        if partition not in self.loaded:
            self.loaded[partition] = json.loads(self.index[partition].read_text('utf-8'))
        return self.loaded[partition]

    def token_estimate(self) -> int:
        return sum(len(json.dumps(v)) // 4 for v in self.loaded.values())
```

## 8. Concurrency Guard (M-016)

```python
class ProjectLock:
    def __init__(self, project_dir: Path):
        self.lock_path = project_dir / '.rag_kernel.lock'

    def acquire(self, session_id: str) -> bool:
        if self.lock_path.exists():
            existing = json.loads(self.lock_path.read_text())
            if existing['session_id'] != session_id:
                return False
        self.lock_path.write_text(json.dumps({
            'session_id': session_id,
            'pid': os.getpid(),
            'acquired_at': datetime.utcnow().isoformat() + 'Z'
        }))
        return True

    def release(self):
        self.lock_path.unlink(missing_ok=True)
```

Split-brain detection: on boot, read HOT `last_checkpoint_seq`. If WAL shows higher seq from different session_id, halt and enter RECOVERY.

## 11. Failure Modes

| Failure | Bridge Behavior | LLM Fallback |
|---------|----------------|--------------|
| Bridge not running | N/A | Autonomous mode (v3.1.x) |
| Crash mid-write | WAL has pre-write entry | WAL replay on restart |
| Lock contention | 409 Conflict + holder info | LLM informs user |
| Hash mismatch | Boot response with errors | RECOVERY per spec |
| Disk full | Write fails, WAL records | RECOVERY, alert user |
| Tunnel disconnect | Bridge still local | GPT loses access, data safe |

## 13. Dual-POV Tension

**ML lens:** Minimize round-trips. Coarse-grained API (batch reads, compound ops). Compact JSON responses.

**CS lens:** Correctness first. Every transition guarded. WAL fsync non-negotiable. Real OS-level locks.

**Resolution:** Both align. Bridge removes enforcement from LLM (fewer self-policing tokens) while providing provable persistence (real fsync, real locks). Tension in API granularity resolved by compound endpoints with granular WAL entries.
